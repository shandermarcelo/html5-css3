Aqui está a segunda Página do exercício 10

 Viu agora você esta na segunda página desse exercício, o link que o seu navegador usou para te trazer até aqui é um Link Interno, no caso o link utilizado por o seguinte:
<a href="pgn-02.html" hreflang="pt-br" rel="next" target="_self">Proxima Página</a>
 Acima está o código de como o link que te trouxe até aqui foi desenvolvido; Primeiro nós temos o elemento representa essa fnção de ligar as coisas o <a> e dentro dele nós temos os parametros href="" que é onde declaramos como valor a URL destino, que no caso de Links internos é uma URL Relativa, esta nós trouxe até essa página. apos esse parametro nós temos o hreflang="", este é um parametro que nos permite indicar para os Mecanismos de Busca qual é o idioma nativo que a página destino utiliza, o que o auxiliara nas questões de tradução.Apos este nós temos o parametro rel="" e aqui nós temos algumas considerações a fazer; Como na HTML5 tudo é desenvolvido com foco em semântica os desenvolvedores devem informar a semântica do seu código sempre que por plausivel e como você sabe quanto mais semântico um código fonte HTML5 for, melhor será para todos e é justamente essa a função do parametro rel, nele nós podemos declarar qual é a semântica dos nossos links. No caso do link que nos trouxe até aqui o valor declarado nele é "next esse é o valor que usamos para indicar que um link é o seguinte, o próximo de uma cadeia de link que pertençem ao mesmo conjunto de conteudo de um site. basicamente informamos que este é um link interno que leva o usuario para a proxima página do mesmo site que ele esta navegando e apos este nós temos o parametro target="" este é um parametro muito importante porque é através dele que nós manipulamos como sera feita a navegação do cliente pelos nossos links. Em caso de link interno o valor que deve ser declarado em target é "_self" este é o valor que nós usamos para indicar que o meio de navegação que sera usado sera "a propria janela atual" ou seja o conteudo do link sera exibido na aba que o cliente esta usando atualmente bascimaente vai substituir o conteudo que o cliente estava interagingo pelo conteudo da URL destino. Mas o nosso exercicio não termina aqui abaixo você tera dois links sendo o primeiro um Link interno que vai te levar devolta para página principal, onde o exercicio começou e sim este link possui declarado como valor "prev" em seu parametro rel="" porque se existe meio de declaramos que o nosso link é o proximo, também podemos declarar qual link é o anterior e é justamente isso que o valor "prev" faz caso seja possivel desenvolver um link de retorno, que volte o cliente para a URL de onde ele partiu sempre declare "prev" no seu parametro rel, isso é semântico e você já sabe a importancia da semântica. "prev" indica que um link é o ANTERIOR ou seja que volta um página e abaixo deste você tera o proximo link desse exercicio
