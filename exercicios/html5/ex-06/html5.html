A Semântica e o HTML5

 A minha conclusão sobre o assunto Semântica e o HTML5 é que, se Semântica é a area de conhecimento que estuda o significado das palavras, então no HTML5 Semântica é o proposito que um elemento cumpre mas o que será isso?
 Esse proposito que eu citei é simplementes A função Objetiva de um elemento por exemplo, os elementos heading, todos eles possuem semântica, que é o significado do que eles fazem, e o que eles fazem, eles contem entre suas tags o conteudo que sera entendido como o tituloe essa função sera compreendida com tal tanto por desenvolvedores que interagirem com o código, quanto pelo o que eu chamo de ©"dispositivos interpretadores" isso engloba Mecanismos de Busca, Tech Assistiva etc.
 Eu entendi que não precisa se preocupar em informar para quem for interagir com o código o que cada elemento que componhe sua estrutura faz, Elementos semanticos já possuem esse valor e o entendimento deles é intricicamente relacionado ao seu valor semântico porém não é só isso; Elementos semânticos possuem também valor organizacional ou seja, um documento HTML5 que é desenvolvido utilizando como pilar principal elementos semânticoé um documento muito mais fácil de se navegar, tanto por humanos(desenvolvedores) quanto pelos ©"Dispositivos Interpretadores" e claro, como o HTML5 foi desenvolvido com foco em seu aspecto Semântico(isso porque o HTML4 tinha um aspecto de forma muito grande, antigamente os elementos eram usados com objetivo em suas formas, que é no efeito que eles trazem) e isso abriu uma janela de oportunidades para que ferramentas como CSS e Javascript pudessem assumir funções para as quais elas foram desenvolvidas e por isso, são muito mais eficientes na aplicação de tais aspectos do que os elementos antigos do HTML por exemplo CSSé uma ferramenta que tem como objetivo implementar manipulações "Estilos" que são estilizações estéticas no HTML então todo o aspecto estético de um documento HTML vai ser desenvolvido através dessa ferramenta, do CSSenquanto ao aspecto interativo do documento, essa parte vai ser desenvolvida através do Javascript.
