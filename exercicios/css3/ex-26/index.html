Grouping Tags?

Grouping Tags, ou Elementos de agrupamento eram elementos que tinham a função de "juntar" o conteudo que eles afiliavam, <div> é um destes elementos, possui seu display como block-level e <span> é outro elemento que possui seu display como inline-level assim, ambos cumpriam a mesma função e eram utilizados para literalmente juntar todo o conteudo de um projeto formando blocos de conteudo agrupado, para diferenciar uma <div> de outra estes elementos possuiam o parametro <id> que como você sabe, tem a função de dar um ideitificador único para o elemento que o possuir e é desse jeito que os elementos de agrupamento funcionam. na HTML 4 pelo menos.

Já na HTML 5 o significado foi introduzido a essencia da linguagem, agora todo conteudo deveria ser desenvolvido visando o significado do mesmo e não mais a forma, e com isso esses elementos de agrupamento "começaram a ficar em desuso" MESMO QUE MUITOS OS USEM ATÉ HOJE Atualmente é recomendado o uso de Elementos Semânticos para fazer o agrupamento de conteudo. se com elementos não semânticos nós tinhamos que desenvolver id para distinguilos uns dos outros agora nós possuimos elementos que possuem seu significado e propósito intrincico a eles, o que facilita na ideitificaçam do conteudo que eles "agrupam" MAS NÃO SE ENGANE AINDA EXISTEM ELEMENTOS QUE DEVEMOS USAR PARAMETROS OU ELEMENTOS ADICIONAIS PARA DISTINGUIR Agora veja um pouco mais sobre estes elementos.

Semântica e Agrupamento

Elementos Semânticos que fazem agrupamento basicamente, são como um elemento <div>, só que semânticos, o que quer dizer que estes elementos possuem em sua essencia um significado que esta relacionado a função que eles cumprem, por exemplo <header> possui o significado de "ser o cabeçalio" e qual é a função desse elemento? SER O DO CONTEUDO QUE ELE AGRUPA! e qual seria esse conteudo agrupado? seria toda parte de um conteudo que estaria afiliada/aninhada a uma <div> que teria seu parâmetro id como "cabeçalio". Essa é a ideia por tras dos elementos semânticos que fazem grouping tags, são elementos que demarcam partes de conteudo sendo que agora, os próprios elementos possuem um significado para si mesmos, um sentido de estarem ali. Agora veja uma lista com alguns elementos semâneticos que são usados para fazer agrupamento

A lista

<header>
Tem a função de ser a <div> do cabeçalio de um conteudo
<nav>
Tem a função de ser um "cardápio" ou menu de navegação. agrupa links
<main>
Tem a função de ser "a área principal" de um conteudo. podendo existir apenas 1 elemento destes por página
<article>
Tem a função de ser "um conteudo independente dos outros" Teriasse um article para cada topico abordado
<aside>
Tem a função de ser conteudo periferico de um <article> como conteudo extra
<section>
Tem a função de ser "parte de um conteudo" Cada sub topico seria uma section, demarcando seu conteudo como um pedaço e dividindo ainda mais o conteudo em áreas separadas
<footer>
Tem a função de ser o rodapé de um conteudo
