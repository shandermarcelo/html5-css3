Elemento <img> e alguns parametros

 <img> é o elemento que você usa para desenvolver conteudo em forma de imagem na HTML5, não se estresse porque esse elemento vai conter a imagem e , no maximo você fara algumas manipulações superficiais nos parametros, e falando de parametros os que foram usados no exercicio são src="" e alt=""
 E agora você me pergunta o que esses parametros fazem? bem o paremetro src=""é onde se declara a URL da imagem, sim tudo está em algum lugar e tudo o que se refere a localização digital de algo se chama URL e é exatamente isso que você declara como valor desse parametro. agora para ser um pouco mais especifico as URLs são categorizadas em dois tipos URL(s) Absoluta e URL(s) Relativa sendo que URL Absoluta é toda URL que NÃO SE LOCALIZA NA DOCUMENTAÇÃO DE UM PROJETO, RESUMIDAMENTE TUDO QUE ESTÁ ALÉM DO CONJUNTO DE PASTAS QUE COMPONHE A DOCUMENTAÇÃO DO QUE VOCÊ ESTÁ DESENVOLVENDO OU DESENVOLVEU É CONSIDERADO UMA URL ABSOLUTA e por isso algumas precauções devem ser tomadas porque como essa URL é de terceiros caso aconteça algum problema lá o recurso(entenda como o que você esta utilizando) que você esta usando pode não ser exibido, o que vai gerar uma imagem de link quebrado contendo apenas o conteudo declarado no parametro alt="" e nós vamos falar dele agora.
 <alt> é um parametro que você vai utilizar para desenvolver um "texto alternativo" como valor, esse texto alternativo é muito importante porque Tecnologias como Leitored de Tela e outras Técnologias assistiva vão utilizar o valor declarado nesse parametro para navegar a página, o que vai dar uma User Experience para pessoas com problemas visuais OS MECANISMOS DE BUSCA TAMBÉM INTERAGEM COM ESSE PARAMETRO PORTANTO O CONTEUDO DESSE TEXTO DEVE SER OBJETIVO MAS PRECISO SEM CONTER INFORMAÇÕES IMPORTANTES.
 Agora VOLTEMOS AS URLs, toda localização digital que EXISTE INTEIRAMENTE NA DOCUMENTAÇÃO DE UM PROJETO É CONSIDERADA URL RELATIVAe essa é a caracteristica marcante desse tipo de URL, se toda sua existencia ou seja o caminho se encontra dentro do conjunto de pastas que componhe a documentação de um projeto então esta é uma URL Relativa; E agora você me pergutna o que isso tem haver com as imagens? bem a primeira imagem ESTA NO MESMO LOCAL ONDE ESTA O ARQUIVO HTML QUE FOI USADO PARA DESENVOLVER O EXERCIO, ISSO FOI USADO PARA EXCLARECER QUE SE PODE USAR UMA IMAGEM QUE ESTEJA NO MESMO LOCAL DO ARQUIVO HTML embora isso não seja a coisa profissional a se fazer, é possivel
 Já a segunda imagem possui uma imagem que esta localizada em uma pasta própria(ESSA É A FORMA PROFISSIONAL DE SE USAR ESSE ELEMENTO) e a ultima imagem possui recurso externo a documentação do projeto então utiliza uma URL Absoluta(e inclusive não esta funcionando mais). Note também que mesmo cada imagem sendo diferente o metodo de desenvolvimento não mudou tanto no parametro alt="" os valores declarados seguiram as mesmas regras, descrições objetivas com toda informação importante para o endendimento do conteudo visual.


Favicon

 Por ultimo mas não menos importante nós temos o nosso Favicon sabe o que é isso? É aquela imagem que esta na frente do nome da página, ISSO É O FAVICON ele geralmente é a logo de um site e sim existe metodos para se criar um favicon e implementalo em um projeto HTML vamos a estes.
 Primeiro vamos tero nosso recurso, este pode ser baixado em sites que possuam favicons QUE POSSUAM LICENÇA PARA USO COMERCIAL LIVRE E MODIFICAÇÃO ou você pode criar seu próprio favicon. não importa como você desenvolva sabia que para fazer uso de um favicon a imagem ou deve ser do formato .ico, de icone ou do formato .png, porque esse formato suporta transparencia ou do formato .svg, que é o melhor formato porem alguns navegadores não possuem compatibilidade com esse formato de arquivo então para evitar problemas use o formato .ico. Com recurso desenvolvido agora basta você implementar no teu projeto, para fazer isso um elemento <link> deve ser desenvolvido na area do elemento <head> isso porque favicon é considerado parte da ideitidade de um site, então ele fica no head e como parametros nós temos rel="" href="" type="" e agora vamos falar um pouco sobre o uso desses parametros para favicon
 rel="" é o parametro onde você vai declarar como valor qual é a relação desse recurso com a página HTML, no caso de favicon você colocara shortcut icon
 href="" é o parametro onde você vai colocar como valor a URL do favicon que você vai usar
 type="" é onde você vai especificar como valor o que é o recurso que esta sendo vinculado a página, basicamente qual é o tipo de recurso; No caso de favicon declare como valor image/x-icon e VOALA! Agora a página possui um favicon.
