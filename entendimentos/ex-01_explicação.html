Essa é a explicação do exercício ex-01, abaixo eu colocarei todo o código desenvolvido e vou explicar o que cada conteudo escrito representa resumidamente mas que dá pra entender

<!DOCTYPE html>
<html lang="pt-br">
<head>
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial,scale=1.0">
<title>Meu Primeiro Exercício</title>
</head>
<body>
<h1>Olá, Mundo!</h1>
<hr>
<p>Esse é um momento único! Estou criando meu primeiro site!</p>
<p>Estou Felizão!</p>
</body>
</html>
 Na primeira linha nós temos o <!DOCTYPE html>, isso é uma declaração de Extensão de Documento, isso sempre deve ser feito antes de qualquer coisa porque tanto os Servidoresquanto os Navegadores precisam saber qual é o tipo de arquivo que eles estão lidando, essa declaração basicamente diz que o arquivo é do tipo HTML5.
 Apos isso nós temos o primeiro elemento que se desenvolve em um projeto HTML, o elemento <html>, html recebe o nome de elemento proque essa nomenclatura engloba todos os componentes do que muitos chamam de tag, tags são os componentes de marcação, representados pelos simbolos de < ou Maior que/Colchete angular e pelos simbolos de > Menor que/Colchete Angular que podem representar ou uma tag de abertura ou uma tag que "fecha em si mesma" que são elementos que não possuem a tag de encerramento, representava pelos simbolos de < Menor que/Colchete angular seguindo de uma / barra da direita para a esquerda, essa barra representa o encerramento e ela é seguida do simbolo de > Maior que/Colchete angular; E este elemento possui um parametro, parametros poderiam ser comparados a acessórios por cumprirem a função de aprimorar a funcionalidade de uma tag, o parametro lang="" possui a função de indicar para os Navegadores que o arquivo que eles receberam possui um idioma nativo, no caso o idioma é o Português Brasileiro e isso além de exclarecer eventuais problemas de comunicação, também vai fazer com que os Navegadores disponibilizem para seus usuários a opção de traduzir o conteudo escrito para o idioma nativo dos usuários; Sabia que o "pt-br" também possui um nome que cataloga dados que estão entre aspas, o nome é Valor e além disso esse valor possui uma especificação é pt-br Português Brasileiro, se o valor fosse apenas "pt" o idioma seria o Português de Portugal, portanto é possivel especificar qual é o idioma exato que o arquivo possui o simbolo de - representa essa especificação; Enquanto a funcionalidade em si do elemento html,este elemento é o que faz o armazenamento de toda a estrutura do código HTML, note que tudo que foi desenvolvido esta entre as tags deste elemento, DOCTYPE não faz parte disso porque ele é uma DECLARAÇÃO, portanto não faz parte da estrutura de código da HTML.
 RESUMÃO: o elemento <html> tem a função de armazenar toda a estrutura de código da HTML, este elemento possui um parametro chamado lang="", que tem a função de indicar para os Navegadores que esse arquivo possui o idioma Português Brasileiro como lingua nativa e tem declarado como valor pt-br que é o valor para Português Brasileiro
 Apos isso nós temos o elemento <head>, esse elemento é responsavel por armazenar todos os elementos que tem funções de configuração de uma página HTML, entenda página como o produto final, como o site ou parte de uma pagina de um site; os conteudos armazenados pelo elemento head NÃO SÃO DE INTERAÇÃO DOS USUARIOS somente os Servidores/Navegadores interagem com o conteudo armazenado pelo elemento head; E pela vasta maioria do conteudo armazenado é de configuração, estes conteudo sequer são de vizualização dos usuários,os poucos que são visiveis para os usuarios são visiveis de formas sutils como o elemento <title>, que tem seu conteudo exibido na aba no navegador, cláro ser o titulo da página não é a única função desse elemento mas para essa explicação visual o que esta dito aqui já é o suficiente para exclarecer como que um usuario pode ver um conteudo que esta nestado no elemento head; O que Nestado significa? Nestar é o efeito de Armazenar um elemento em outro, o elemento <html> nesta os elementos <head> e <body> estes por sua vez também estão nestando elementos, portanto esse ato de armazenar é conhecido como nestar, aninhar uma coisa em outra, um dentro do outro
 Enquanto ao conteudo nestado nós temos elementos <meta>, esse elemento são um dos tipos de dados que se usa no elemento <head>, seu conteudo representa um metadado, que é um dado de configuração e note que o elemento meta NÃO POSSUI TAG DE ENCERRAMENTO, isso é proque não são todos os elementos da HTML que possuem tags de fechamento alguns não possuem e o elemento <meta> é um destes elementos. Mas ai você se questiona

"Se um elemento não possui tag de encerramento como que se desenvolve conteudo nele?"
A resposta para isso é simples, Elementos que não possuem tag de encerramento tem seu conteudo desenvolvido através de parametros, alguns elementos possuem parametros especificos para eles mesmos "parametros nativos" e muitos possuem "parametros universais" que são parametros que todo elemento pode fazer uso e no nosso caso os elementos meta estão usando parametros distintos o primeiro elemento meta esta usando um parametro chamado charset="" esse parametro tem a função de informar para os Navegadores/Servidor que esse arquivo é compátivel com o conjunto de caracteres, e o valor declarado é unicode "UTF-8" e esse valor é muito útil porque esse conjunto de caracteres possui suporte para acentuaçõeso que é necessário na web atual. O segundo elemento meta utiliza dois parametros primeiro é o parametro name="", esse parametro tem a função de ser o ideitificador do tipo de dado que sera o conteudo do elemento meta, no caso o valor declarado é "viewport", este valor é o ideitificador de que o conteudo do elemento meta será relacionado a adaptação da resolução de tela e o segundo parametro é o content="", como o nome indica esse parametro possui como valor justamente o conteudo do metadado em si, no caso o valor declarado é "width=device-width, initial-scale=1.0" e o que que isso tudo quer dizer? isso quer dizer que a resolução de tela do arquivo HTML ira ocupar 100% da extensão do elemento <body> ou seja a resolução de tela do arquivo HTML ira se adaptar por toda a extensão da página, isso vai fazer comque caso o cliente(usuário) esteja utilizando um disposivito móvel(smartphone) ou um Tablet enfim, um dispositivo que não seja compativel com o dispositivo onde o arquivo HTML foi desenvolvido e esteja acessando esse arquivo.A RESOLUÇÃO DE TELA DO ARQUIVO HTML VAI SE ADAPTAR A RESOLUÇÃO DO DISPOSITIVO DO CLIENTE A 100%, ZEM USO DE ZOOM E OUTROS EFEITOS DE RESOLUÇÃOisso fará com que a pagina se torne Responsiva, que é adaptavel a outros dispositivos.
 E por ultimo nós temos o elemento <title> Esse elemento tem a função de conter o conteudo que será o titulo da página, esse titulo vai ser exibido na Aba, também conhecida como "Janela" do Navegador; Esse titulo pode ser exibidos nos resultados das pesquisa pelos Mecanismos de Busca, que é o local onde os clientes fazem a busca pelas URLS dos sites que eles desejam interagir.


 Agora nós vamos para a area do elemento <body>, Esse elemento também nesta elementos porém os que são nestados aqui são a parte do conteudo da página, ou seja tudo que o cliente vai ver e interagir na página é aninhado por esse elemento. e o primeiro elemento que nós temos aninhado é o elemento <h1> esse elemento possui o nome de heading1 ou heading level 1, ea função dele é de ser o titulo principal do conteudo de uma página, por ser um elemento que tem a função de ser o principal ele além de possui uma Hierarquia, também possui Semântica, mas qual é essa hirarquia? Bem essa hirarquia é composta por elemento heading, que vão de <h1> e vão até <h6> sendo que a ordem de importancia na hierarquia é decresente, ou seja quanto menor o número do elemento heading maior será sua importancia nessa hierarquiaSabia que todos os elementos heading podem ser utilizados mas os mais utilizados são <h1> até o <h3> TENTE ENTENDER O <h1> COMO TITULO PRINCIPAL, TITULO DO TEMA DA PÁGINA <h2> COMO TITULO DE TOPICO E <h3> COMO TITULO DE SUBTOPICO; Outra coisa importante a respeito da hierarquia é que O ELEMENTO <h1> POR SER O QUE ESTA NO TOPO DA HIERARQUIA E POSSUIR A FUNÇÃO DE SER O TITULO PRINCIPAL SÓ PODE SER UTILIZADO 1 ÚNICA VEZ POR PÁGINA, OS OUTROS HEADINGS PODEM SER REUTILIZADOS QUANTAS VEZES QUISER.Enquanto a Semantica isso é um termo que significa Ter Sentido, Ser compreesivo para a "Máquina" entenda máquina como o dispositivo computácional em si, como se sabe Computadores não compreendem idiomas, eles compreendem apenas o Binarismo ou seja 0 para negativo 1 para positivo e a semantica tem essa função de fazer com que a máquina possa comrpeender o que um código representa na sua linguágem, muito da Programação tem haver com ESCREVER CÓDIGO QUE HUMANOS ENTENDEM E TRADUZIR ESSES MESMOS CÓDIGOS PARA QUE AS MÁQUINAS ENTENDAMe não só os elementos heading, como muitos outros possuem esse aspecto semanticoinclusive o proximo elemento do nosso exemplo que é o elemento <hr> esse elemento tem a função de indicar tanto para o cliente quanto para a máquinaELE POSSUI SEMANTICA que houve uma mudança de conteudo. O proximo e ultimo conteudo do nosso exemplo é o elemento <p> Esse elemento tem a função de conter conteudo textual, representa um paragrafo, sim ele tem semanticaE sabia que esse elemento não possibilita que se demarque onde um paragrafo começa e onde ele termina. Para isso nós teremos que usar outros elementos mas isso fica para a proxima. :)
