Links e o Cliente

 Como você já deve ter percebido na Era da Informaçãoatrair a atenção das pessoas e mantelas interagindo com o conteudo se tornou essêncial para a sobrevivência de quem produz conteudo para Internet e como se sabe os Links são uma parte muito imporante disso e é justamente pensando nisso que SE DEVE CONSILIAR O BOM DESENVOLVIMENTO DE LINKS COM A INTERATIVIDADE DO(S) CLIENTE(S) COMA SUA APLICAÇÃOe é ai que vem uma estratégia para o desenvolvimento de Links.
 Como um Link Externo vai levar o usuario para outra aplicação PROVAVELMENTE ELE NÃO VAI VOLTAR MAIS PARA ONDE ELE ESTAVA porque como cada local da internet esta repleto de informações e conteudos que visam fisgar a atenção e o interesse dos usuarios então dificilmente quando um usuario sai de um endereço e vai para outro ele retornará para onde ele estava.Especialmente com Links Externos porque o uso de um link de retorno NÃO SERÁ POSSIVEL NA MAIORIA DAS VEZES e é visando uma consiliação que alguns parametros são usados para garantir que todos saiam ganhando veja eles abaixo:
href=""~Esse é o parametro usado para declarar a "destinação" que seria o endereço de para onde o usuario será transportado, esse endereço se é conhecido com URL e estas podem ser categorizadas em URL Absoluta, que é todo endereço que for externo a Documentação de uma aplicação e temos as URL Relativa, que é todo endereço que pertençe a Documentação de uma Aplicação e é justamente esse endereço que nós declaramos como valor nesse parametro
hreflang="" é um parametro que tem a função de indicar para os mecanimos de busca/navegadores qual é o idioma que a URL declarada possui, isso os auxilia com questões de tradução, o valor declarado nesse parametro é identico ao que se declara no parametro lang="" os seja são as inicais do idioma
target=""~Este é um parametro que tem a função de "manipular o método de transporte de uma area clicável" e é aqui onde nós usamos nossa estratégia de consiliação aqui no caso de Links Externos nós declaramos como valor "_blank" este valor determina que o link do elemento <a> que o possua em seu parametro href abra o link em uma nova Aba em caso de interação isso vai fazer que o usuario não perca totalmente o acesso a página de onde ele saiu, mesmo ele tendo ido para outro local, com o endereço da URL Absoluta aberto em uma nova Janela o usuario tera a chance de voltar para a página d eonde ele saiu utilizando as janelas. É POR ISSO QUE O VALOR "_blank" DEVE SER USADO PARA LINKS EXTERNOS
rel="" é um parametro que usados para indicar para mecanismos de busca/navegadores como um recurso se relaciona a nossa página, nesse caso o recurso é um link e a relação desse recurso com a nossa página é a de uma URL Absoluta ou seja é a relação de um LINK EXTERNO portanto declaramos como valor "external". SEMPRE DECLARE "external" como valor nesse parametro quando for desenvolver links que usam URLs Absolutas
